# 基于 Spring Boot 的图书管理系统需求文档

## 1. 项目概述

随着图书馆图书数量和读者人数不断增加，传统人工登记或 Excel 表格管理方式已经难以满足日常管理需求。图书借阅、归还、库存统计、读者信息维护等工作如果完全依赖人工完成，不仅效率较低，而且容易出现信息遗漏、库存不准确、借阅记录混乱等问题。

本项目开发一套基于 Spring Boot 的图书管理系统，实现图书信息、用户信息、图书分类和借阅记录的统一管理。系统主要面向学校图书馆或小型图书室，帮助管理员提高图书管理效率，也方便普通用户查询和借阅图书。

## 2. 项目名称

基于 Spring Boot 的图书管理系统

## 3. 技术架构

本系统采用前后端分离架构，后端负责业务处理和数据接口，前端负责页面展示和用户操作。

## 4. 系统用户角色

本系统主要包括两类用户：管理员和普通用户。

### 4.1 管理员

管理员负责系统后台管理工作，主要对用户信息、图书分类、图书信息和借阅记录进行维护。

### 4.2 普通用户

普通用户主要指学生或读者，可以登录系统查询图书、借阅图书、归还图书，并查看自己的借阅记录。

## 5. 功能性需求

## 5.1 登录模块

系统提供用户登录功能。用户输入用户名和密码后，系统对账号信息进行校验。

登录成功后，系统根据用户角色进入不同页面。管理员进入后台管理页面，普通用户进入图书查询页面。

### 功能要求

| 功能   | 说明               |
| ---- | ---------------- |
| 用户登录 | 用户输入用户名和密码登录系统   |
| 身份判断 | 判断当前用户是管理员还是普通用户 |
| 退出登录 | 用户可以退出当前账号       |
| 访问控制 | 未登录用户不能进入系统页面    |

## 5.2 用户管理模块

用户管理模块主要供管理员使用。管理员可以对系统中的普通用户信息进行管理。

### 功能要求

| 功能   | 说明            |
| ---- | ------------- |
| 新增用户 | 添加普通用户账号      |
| 修改用户 | 修改用户姓名、手机号等信息 |
| 删除用户 | 删除无效用户信息      |
| 查询用户 | 按用户名或姓名查询用户   |
| 查看用户 | 查看用户基本信息      |

### 用户信息内容

用户信息主要包括用户编号、用户名、密码、真实姓名、手机号、用户角色、账号状态、创建时间等。

## 5.3 图书分类管理模块

图书分类管理模块用于维护图书分类信息，方便管理员对图书进行分类管理，也方便普通用户按照分类查询图书。

### 功能要求

| 功能   | 说明        |
| ---- | --------- |
| 新增分类 | 添加新的图书分类  |
| 修改分类 | 修改分类名称和描述 |
| 删除分类 | 删除无效图书分类  |
| 查询分类 | 查看所有图书分类  |

### 分类示例

| 分类名称 | 说明                |
| ---- | ----------------- |
| 计算机类 | Java、数据库、计算机网络等图书 |
| 文学类  | 小说、散文、诗歌等图书       |
| 历史类  | 历史文化相关图书          |
| 教育类  | 教育教学相关图书          |
| 经济类  | 经济管理相关图书          |

## 5.4 图书信息管理模块

图书信息管理是系统的核心功能，主要由管理员使用。管理员可以对图书馆中的图书信息进行添加、修改、删除和查询。

### 功能要求

| 功能   | 说明                     |
| ---- | ---------------------- |
| 新增图书 | 添加图书名称、作者、出版社、分类、库存等信息 |
| 修改图书 | 修改已有图书的基本信息            |
| 删除图书 | 删除无效或下架图书              |
| 查询图书 | 按书名、作者、分类查询图书          |
| 查看详情 | 查看图书详细信息               |
| 库存管理 | 维护图书总数量和可借数量           |

### 图书信息内容

图书信息主要包括图书编号、图书名称、作者、出版社、图书分类、总数量、可借数量、图书状态、创建时间等。

## 5.5 图书查询模块

普通用户登录系统后，可以查询图书信息，了解图书是否可以借阅。

### 功能要求

| 功能     | 说明           |
| ------ | ------------ |
| 查看图书列表 | 查看系统中的图书信息   |
| 书名查询   | 根据图书名称进行模糊查询 |
| 作者查询   | 根据作者名称查询     |
| 分类查询   | 根据图书分类筛选     |
| 查看详情   | 查看图书详细信息     |
| 显示库存   | 显示图书当前可借数量   |

## 5.6 图书借阅模块

普通用户可以对库存充足的图书进行借阅。

### 借阅规则

| 规则     | 说明                 |
| ------ | ------------------ |
| 用户必须登录 | 未登录用户不能借阅图书        |
| 图书必须可借 | 图书可借数量必须大于 0       |
| 不能重复借阅 | 同一用户不能重复借阅同一本未归还图书 |
| 默认借阅时间 | 默认借阅期限为 30 天       |

### 功能要求

| 功能   | 说明           |
| ---- | ------------ |
| 借阅图书 | 用户选择图书后进行借阅  |
| 生成记录 | 借阅成功后生成借阅记录  |
| 减少库存 | 图书可借数量自动减少 1 |
| 显示状态 | 借阅记录状态显示为未归还 |

## 5.7 图书归还模块

普通用户可以对已经借阅的图书进行归还操作。

### 功能要求

| 更新记录   | 借阅记录状态修改为已归还    |
| 记录时间   | 系统保存实际归还时间      |

## 5.9 个人信息模块

普通用户可以查看和修改自己的个人信息。

### 功能要求

| 功能     | 说明              |

## 6. 非功能性需求

## 6.1 易用性需求

系统页面应简洁清晰，功能菜单分类明确。管理员和普通用户登录后显示不同的功能菜单，用户能够快速找到需要使用的功能。

## 6.2 安全性需求

系统应具有基本的登录验证功能。管理员和普通用户权限需要区分，普通用户不能访问管理员管理页面。

用户密码不建议直接明文保存，可以使用简单加密方式进行保存。

## 6.3 稳定性需求

系统应能够稳定完成登录、查询、新增、修改、删除、借阅和归还等基本操作。对于输入错误或操作失败的情况，系统应给出提示信息。

## 6.4 可维护性需求

后端代码采用分层结构开发，包括 Controller 层、Service 层、Mapper 层和实体类层。代码命名应清晰规范，方便后续维护和修改。

## 6.5 数据一致性需求

系统需要保证图书库存和借阅记录保持一致。用户借阅图书成功后，图书可借数量减少；用户归还图书成功后，图书可借数量增加。

## 7. 系统业务流程

## 7.1 登录流程

用户进入登录页面。

用户输入用户名和密码。

系统校验账号和密码。

校验成功后判断用户角色。

管理员进入后台管理页面。

普通用户进入图书查询页面。

## 7.2 图书借阅流程

用户登录系统。

用户查询图书信息。

用户选择需要借阅的图书。

系统判断图书是否有库存。

如果图书可以借阅，系统生成借阅记录。

图书可借数量减少 1。

系统提示借阅成功。

## 7.3 图书归还流程

用户登录系统。

用户进入个人借阅记录页面。

用户选择未归还图书。

用户点击归还按钮。

系统修改借阅记录状态。

系统记录归还时间。

图书可借数量增加 1。

系统提示归还成功。

## 12. 项目实现范围

本系统作为期末大作业，主要完成以下核心功能：

用户登录。

管理员管理用户。

管理员管理图书分类。

管理员管理图书信息。

普通用户查询图书。

普通用户借阅图书。

普通用户归还图书。

管理员查看所有借阅记录。

普通用户查看个人借阅记录。

本项目不实现复杂罚款计算、短信提醒、图书推荐、Excel 导入导出和第三方登录等功能。这些功能可作为后续优化方向。
